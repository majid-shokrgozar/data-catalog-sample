# Documentation Standards

This document outlines the coding standards for documentation in Java and .NET projects.

## Java Documentation Standards

1. **JavaDoc Comments**: 
   - Use `/** ... */` for class and method documentation.
   - Every public class and method should have documentation comments.
   - Include parameters, return values, and possible exceptions in the comments.

2. **Naming Conventions**:
   - Class names should be nouns and in CamelCase (e.g., `MyClass`).
   - Method names should be verbs in camelCase (e.g., `myMethod`).
   - Constants should be in uppercase with underscores (e.g., `MY_CONSTANT`).

3. **Code Style**:
   - Indentation should be 4 spaces.
   - Use meaningful names for variables and methods.
   
## .NET Documentation Standards

1. **XML Documentation Comments**: 
   - Use `///` for class and method documentation.
   - Every public class and method should have XML comments.
   - Include `<param>`, `<returns>`, and `<exception>` tags in the comments.

2. **Naming Conventions**:
   - Class names should be nouns and in PascalCase (e.g., `MyClass`).
   - Method names should be verbs in PascalCase (e.g., `MyMethod`).
   - Constants should be in PascalCase (e.g., `MyConstant`).

3. **Code Style**:
   - Indentation should be 4 spaces.
   - Use meaningful names for variables and methods.
